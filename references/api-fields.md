# 實價登錄成交欄位與單位規則

本文件說明查詢結果與分析結果使用的欄位。產生統計或報告時，優先使用 `analyze-transactions.js` 產生的正規化欄位；不要再次換算已正規化的數值。

## 原始成交欄位

| 欄位 | 意義 | 使用規則 |
|---|---|---|
| `a` | 地址或區段位置 | 只顯示官網實際回傳內容，不補寫隱碼門牌 |
| `bn` | 社區或大樓名稱 | 空值時顯示 `—` |
| `city` | 縣市代碼 | 與 `/SERVICE/CITY` 回傳資料核對 |
| `town` | 鄉鎮市區代碼 | 通常為縣市代碼加數字代碼 |
| `e` | 交易日期 | 通常為民國年 `YYY/MM/DD` |
| `tp` | 成交總價 | 依 `tunit` 或程式正規化結果判定單位 |
| `p` | 成交單價 | 依 `unit`、`punit` 或程式正規化結果判定單位 |
| `s` | 建物移轉面積 | 依 `unit` 判定為坪或平方公尺 |
| `b` | 建物型態 | 例如公寓、華廈、住宅大樓、透天厝或套房 |
| `g` | 屋齡 | 本專案分析程式使用的主要屋齡欄位 |
| `f` | 成交樓層及總樓層 | 保留官網原始文字供顯示與比較 |
| `v` | 格局 | 空值時顯示 `—` |
| `t` | 交易標的 | 用於辨識房地、土地與車位組合 |
| `cp` | 車位總價 | 僅在數值及單位可確認時使用 |
| `note` | 特殊交易備註 | 用於特殊交易、增建及排除原因判斷 |
| `unit` | 面積及單價單位代碼 | 本專案已驗證 `1` 與 `2` 的換算規則 |
| `punit` | 單價單位代碼 | 作為輔助驗證，不覆蓋已正規化結果 |
| `tunit` | 總價單位代碼 | 作為輔助驗證，不覆蓋已正規化結果 |
| `msg` | 官網單價計算說明 | 若存在，可用來判斷單價是否已排除車位 |
| `el` | 是否有電梯 | 保留原始文字 |
| `m` | 是否有管理組織 | 保留原始文字 |
| `pu` | 主要用途 | 用於判斷住宅市場用途是否相符 |
| `lat`、`lon` | 座標 | 只在本次任務確有需要時使用 |
| `commid`、`sq` | 內部識別或雜湊欄位 | 可輔助對應案例，不直接向使用者解釋其業務意義 |

不得把 `r` 當作屋齡。本專案以 `g` 及正規化後的 `_age` 為準。

## 單位換算

### `unit = 2`

- `s` 已是坪。
- `p` 已是元／坪。
- 不得再次將 `s` 除以 `3.305785`。
- 不得再次將 `p` 乘以 `3.305785`。

換算為萬元／坪：

```text
萬元／坪 = 元／坪 ÷ 10,000
```

### `unit = 1`

- `s` 為平方公尺。
- `p` 為元／平方公尺。

換算公式：

```text
坪數 = 平方公尺 ÷ 3.305785
元／坪 = 元／平方公尺 × 3.305785
萬元／坪 = 元／坪 ÷ 10,000
```

若 `unit`、`punit`、`tunit` 缺少、互相矛盾或出現未知代碼，不得猜測單位。將案例標示為單位無法確認，並依分析程式輸出決定是否排除。

## 正規化欄位

正式報告優先使用：

| 欄位 | 意義 |
|---|---|
| `_areaPing` | 正規化後面積（坪） |
| `_pricePerPing` | 正規化後單價（元／坪） |
| `_pricePerPingWan` | 正規化後單價（萬元／坪） |
| `_age` | 正規化後屋齡 |
| `_hardFlags` | 重大特殊交易標記 |
| `_additionSeverity` | 增建嚴重程度 |
| `_additionKeywords` | 增建關鍵字 |
| `_exclusionReasons` | 排除原因 |

不要對 `_areaPing`、`_pricePerPing`、`_pricePerPingWan` 再做換算。

## 車位價格

若 `msg` 明確顯示單價已使用「總價減車位總價、總面積減車位面積」計算，直接使用正規化單價，不要重算。

只有在車位總價、車位面積、數值單位及欄位意義均可確認時，才自行拆分：

```text
住宅單價 =（房地總價 − 車位總價）÷（建物總面積 − 車位面積）
```

若交易包含車位但無法可靠拆分，保留案例並附註：

```text
本筆交易包含車位，但車位價格或面積無法可靠拆分，所列單價可能受到車位影響。
```

## 日期換算

`e` 若為民國年格式：

```text
西元年 = 民國年 + 1911
```

報告可保留民國年原始日期；若轉換為西元年，必須驗證日期有效。

## 未確認欄位

未經程式或資料樣本確認的欄位不得用於統計，也不得斷言其意義。若必須顯示，只能保留原值並標示「欄位意義待確認」。
